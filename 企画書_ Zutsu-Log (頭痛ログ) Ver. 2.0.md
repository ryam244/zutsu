# 企画書: Zutsu-Log (頭痛ログ) Ver. 2.0

**作成日:** 2026年1月20日
**作成者:** Manus AI

---

## 1. エグゼクティブサマリー

Zutsu-Logは、気圧の変化と個人の体調（特に頭痛）の相関関係を解明し、「痛みの予報」と「シンプルな記録」に特化したヘルスケア×天気アプリケーションです。本企画は、既存の天気アプリが提供する一般的な気象情報ではなく、**「いつ頭痛が来るか」「なぜ今痛いのか」**というユーザーの切実なペイン（Pain）に対して、明確かつ直接的なソリューションを提供することを目的とします。競合分析と市場調査に基づき、Zutsu-Logは「多機能だが複雑な既存アプリ」と「記録は詳細だが予測が弱いアプリ」の間に存在する明確な市場機会を捉え、**「シンプルさ」と「予測精度」**を両立させることで独自のポジションを確立します。初期段階では、コア機能である「気圧低下アラート」と「ワンタップ記録」にリソースを集中させ、ミニマムなUXで熱量の高い初期ユーザー層を獲得し、段階的に収益化と機能拡張を目指す戦略を提案します。

## 2. プロダクトコンセプト

### 2.1. Vision

> 「天気に振り回される人生を、コントロール可能なものに変える」

このビジョンは、ユーザーが自身の体調をより深く理解し、予測不能な痛みに対して主体的に対策を講じられるようになる世界を目指すものです。

### 2.2. ターゲットオーディエンス

市場調査の結果を踏まえ、ターゲットオーディエンスをより明確に定義し直します。

| ターゲット層 | 詳細 | 市場規模（推定） |
| :--- | :--- | :--- |
| **プライマリターゲット** | 20代～40代の女性。偏頭痛と気象病（天気痛）の両方に悩まされており、スマートフォンでの情報収集と自己管理に積極的。 | 約840万人 [1] |
| **セカンダリターゲット** | 偏頭痛、緊張型頭痛、または低気圧による体調不良（だるさ、めまい等）に悩む全ての性別・年代。 | 約1,000万人 [2] |

初期フェーズではプライマリターゲットに特化し、彼女たちの切実なニーズに応えることで、プロダクトのコア価値を確立します。

### 2.3. 独自の価値提案 (Unique Value Proposition)

競合製品との差別化を図るため、以下の3つの価値を明確に定義します。

1.  **ペイン・ファーストUI (Pain-First UI)**: 天気予報ではなく「痛み予報」を最優先に表示します。「頭痛ーる」のような一般的な天気アプリとは一線を画し、ユーザーが最も知りたい情報に最短でアクセスできる体験を提供します。
2.  **究極のシンプルさ (Radical Simplicity)**: 起動から記録完了まで1秒。思考停止状態でも直感的に使えるUI/UXを徹底します。これは、詳細な記録を求める「Migraine Buddy」とは対照的なアプローチであり、記録の負担を最小限に抑えたいユーザー層に訴求します。
3.  **納得感によるメンタルケア (Mental Relief through Confirmation)**: 「やはり気圧のせいだった」という原因の可視化は、ユーザーに大きな安心感と納得感を与えます。これは単なる記録ツールではなく、痛みの原因が不明であることの不安を和らげるメンタルケアツールとしての価値を提供します。

## 3. 市場分析とプロダクトのポジショニング

### 3.1. 市場規模と機会

日本の偏頭痛患者数は約840万人、気象病の潜在患者数は1,000万人を超えると報告されています [1][2]。特に、頭痛の要因として「気圧の変化」を挙げる人は55.5%に上り、気圧予報と体調管理を結びつけるアプリケーションには大きな需要が存在します [3]。一方で、既存のソリューションは全てのユーザーニーズを満たせておらず、特に「シンプルさ」と「予測」を両立する領域に市場機会が存在します。

### 3.2. 競合分析

| 項目 | **Zutsu-Log (本企画)** | **頭痛ーる** | **Migraine Buddy** |
| :--- | :--- | :--- | :--- |
| **ターゲット** | 気象病・偏頭痛の重複層 | 気象病全般 | 偏頭痛 |
| **コアバリュー** | **痛み予報 + シンプル記録** | 気圧予報 + 多機能 | 詳細記録 + コミュニティ |
| **UI/UX** | ペイン・ファースト、ミニマル | 天気アプリ型、情報過多 | 医療記録型、入力項目多 |
| **気象連携** | **高 (2時間前アラート)** | 高 (1時間単位グラフ) | 低 (7日間予報) |
| **医療連携** | フェーズ2で計画 | 限定的 | 高 (医師へのレポート共有) |
| **ユーザー数** | - | 2000万DL以上 | 350万 (グローバル) |

### 3.3. 戦略的ポジショニング

Zutsu-Logは、**「頭痛ーるの予測機能」**と**「Migraine Buddyの患者視点」**の長所を組み合わせ、両者が満たせていない**「究極のシンプルさ」**という価値で市場に参入します。複雑な機能や詳細な入力を求めるのではなく、「鎮痛剤を飲むタイミングを知りたい」という最もクリティカルなニーズに一点集中することで、独自のポジションを築きます。

## 4. MVP (Minimum Viable Product) 機能要件

「予測」と「記録」のコア体験を最小限の機能で実現することにリソースを集中させます。

| カテゴリ | 機能名 | 詳細要件 | 優先度 |
| :--- | :--- | :--- | :--- |
| **予測 (Predict)** | 気圧低下アラート | 設定地点の気圧が急激に低下する「2時間前」などにプッシュ通知を送信する。通知タイミングは設定可能。 | **High** |
| **予測 (Predict)** | 気圧グラフ表示 | 直近24時間と今後24時間の気圧変動をグラフ化。「爆弾マーク（警戒）」などで視覚的に警告する。 | **High** |
| **記録 (Log)** | ワンタップ体調記録 | 4段階（なし/少し/痛い/寝込む）のアイコンタップのみで記録完了。メモ機能は任意とする。 | **High** |
| **設定 (Config)** | 地点登録 | GPS常時追跡は行わず、都道府県または市町村単位での固定地点登録（最大3ヶ所）とする。 | **Medium** |
| **分析 (Analyze)** | 記録プロット表示 | 気圧グラフ上にユーザーの記録アイコンを重ねて表示し、相関を視覚化する。 | **Medium** |

**【提案】** 当初の「分析」機能の優先度を`Medium`に下げ、MVPリリース後のユーザーフィードバックに基づき実装を検討します。これにより、MVPのシンプルさをさらに際立たせることができます。

## 5. 技術アーキテクチャとリスク管理

提案されている技術スタックは、迅速な開発とスケーラビリティの観点から適切です。

- **Frontend**: React Native (Expo SDK 50+)
- **Backend**: Supabase
- **API**: OpenWeatherMap (One Call API 3.0)

### 5.1. コスト最適化戦略

Edge Functionsを用いた「プロキシキャッシュ戦略」は、APIコストを抑制する上で極めて有効であり、本企画の収益性を担保する重要な要素です。この戦略を前提として開発を進めます。

### 5.2. 技術的リスクと対策

| リスク | 内容 | 対策 |
| :--- | :--- | :--- |
| **プッシュ通知の信頼性** | Expoのプッシュ通知サービスは設定が複雑で、到達率が環境に依存する可能性がある。 | 開発初期段階で通知機能の技術検証（PoC）を実施し、必要であれば代替サービス（OneSignal等）を検討する。 |
| **気圧データの精度** | OpenWeatherMapのデータと実際の体調変化の相関性が、必ずしも全てのユーザーに当てはまるとは限らない。 | アプリ内で「本データは予報であり、医学的診断ではない」ことを明記する。将来的には複数の気象データソースを組み合わせることも検討。 |
| **データプライバシー** | 体調記録は機微な個人情報であり、厳格な管理が求められる。 | SupabaseのRLS（Row Level Security）を徹底活用し、ユーザー本人以外はデータにアクセスできない設計を保証する。プライバシーポリシーを明確に策定し、公開する。 |

## 6. Go-to-Market戦略 (初期ユーザー1,000人獲得)

オーガニックな成長戦略に加え、より多角的で能動的なアプローチを組み合わせます。

1.  **ASO (App Store Optimization) の徹底**: 「頭痛」「低気圧」「気圧」「偏頭痛」「自律神経」「天気痛」などのキーワードで検索上位表示を目指します。
2.  **SNSでのゲリラマーケティング (X/Twitter)**: 「爆弾低気圧」到来時に、気圧グラフのスクリーンショットと共に注意喚起を行う戦術は有効です。これを自動化するBotを開発し、継続的に実施します。
3.  **コミュニティへの直接アプローチ**: 偏頭痛や気象病に関するオンラインコミュニティ（Xの関連ハッシュタグ、LINEオープンチャット等）で、ベータ版テスターを募集し、プロダクトの初期ファンを獲得します。
4.  **医療・ウェルネスメディアへのPR**: 気象病をテーマにした記事や特集を組んでいるウェブメディアに対し、情報提供やアプリ紹介のプレスリリースを送付します。

## 7. 将来のロードマップと収益化

| フェーズ | 期間（目安） | 主要目標 | 収益化 |
| :--- | :--- | :--- | :--- |
| **Phase 1 (MVP)** | 3ヶ月 | 初期ユーザー1,000人獲得、PMF（プロダクトマーケットフィット）の検証 | なし |
| **Phase 2 (収益化)** | 6ヶ月 | ユーザー数10万人、プレミアム機能導入 | **プレミアムプラン (月額300円程度)**<br>- 広告除去<br>- 複数地点登録（5ヶ所以上）<br>- 過去データの長期閲覧 |
| **Phase 3 (拡張)** | 12ヶ月 | ユーザー数100万人、医療連携 | **B2B2Cモデルの検討**<br>- 医療機関向けダッシュボード提供<br>- 企業向け健康経営ソリューション |

**【提案】** 当初の月額200円という価格設定は、市場の類似アプリ（例：頭痛ーるプレミアムは約360円）と比較して安価すぎる可能性があります。付加価値を明確にした上で、月額300円程度からの開始を推奨します。
